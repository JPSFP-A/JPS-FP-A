### 1. `doLogin is not defined` (Root Cause: SyntaxError)

**Symptom:** `doLogin is not defined` on page load вЂ” no functions available at all.

**Root cause:** An extra `}` in the `mkChart('cDep', ...)` call on JS line 8004 caused the entire script block to fail parsing. The line had 17 `{` opens and 19 `}` closes (net в€’2; expected в€’1).

**Diagnosis approach:**
- Binary search through the script to isolate failing half
- Extracted JS block with `node --check`
- Built a delimiter-counting Node.js script (`count_line8004.js`) to scan the specific line character-by-character, accounting for string context
- Found `BRACE NEGATIVE at pos 489` вЂ” one extra `}` in the closing sequence

**Fix вЂ” HTML line 12016:**
```js
// BEFORE (broken вЂ” 6 closing braces before closing paren):
mkChart('cDep',{...options:{...scales:{...}}}}}});

// AFTER (fixed вЂ” 5 closing braces):
mkChart('cDep',{...options:{...scales:{...}}}}});
```

### 2. `RangeError: Maximum call stack size exceeded`

**Symptom:** After login succeeded, the app crashed immediately with infinite recursion at `_acts`.

**Root cause:** `_acts` called itself via the `??` fallback operator:
```js
const _acts = (mo) =>
  actualsStore[actualsYear]?.[mo] ??
  (actualsYear === 2026 ? _acts(mo) : undefined);  // в†ђ calls itself
```
When `actualsStore[2026]?.[mo]` was `undefined`, the fallback triggered `_acts(mo)` again вЂ” infinite loop.

**Fix вЂ” HTML line ~7468:**
```js
// BEFORE (broken вЂ” self-referential):
const _acts = (mo) =>
  actualsStore[actualsYear]?.[mo] ??
  (actualsYear === 2026 ? _acts(mo) : undefined);

// AFTER (fixed вЂ” simple direct lookup):
// Returns the actuals value for month `mo` under the active actualsYear.
// actualsStore is keyed { year: { mo: value } } for all years.
const _acts = (mo) => actualsStore[actualsYear]?.[mo];
```

### 3. Sales Platform Login Broken After FP&A Login

**Symptom:** Logging into FP&A broke the Sales Platform session вЂ” users had to log in twice on every switch.

**Root cause:** Both apps used Supabase's default localStorage key (`sb-<project>-auth-token`). The FP&A app overwrote the Sales Platform's session token.

**Fix вЂ” HTML line ~4046:** Added `storageKey` and `detectSessionInUrl: false` to the FP&A Supabase client:
```js
// BEFORE (no storageKey вЂ” collides with Sales Platform):
_sb = window.supabase.createClient(_sbConfig.url, _sbConfig.anonKey, {
  auth: { persistSession: true, autoRefreshToken: true },
  realtime: { params: { eventsPerSecond: 10 } }
});

// AFTER (isolated session namespace):
_sb = window.supabase.createClient(_sbConfig.url, _sbConfig.anonKey, {
  auth: {
    persistSession: true,
    autoRefreshToken: true,
    storageKey: 'jps_fpa_v1',      // isolates FP&A session from Sales Platform
    detectSessionInUrl: false,      // prevents grabbing tokens meant for other apps
  },
  realtime: { params: { eventsPerSecond: 10 } }
});
```
