📋 Step 1 — Select a Version

Choose the planning version this debt data belongs to
How this works The debt schedule is tied to a planning version (e.g. March 2026 LE, 2026 Budget). Each upload replaces all debt data for that version — existing facilities and schedule rows are deleted and re-created. Actuals versions record the actual debt position at month-end.

📥 Step 2 — Download the CSV Template

Fill in your debt data, one row per facility × month
CSV structure — one row per Facility × Month Each loan facility is identified by a stable facility_code (e.g. NCB_JMD). Repeat facility metadata on every row — the uploader deduplicates automatically. All monetary values should be in the facility's native currency (JMD, USD or EUR). Provide fx_rate (local currency per USD) so the platform can convert to USD.
