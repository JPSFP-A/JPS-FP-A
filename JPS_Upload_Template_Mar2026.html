KEY DIFFERENCES: Budget Upload vs. Actuals Upload

DimensionActuals UploadBudget Upload (AOP)
RevenueRead directly from Oracle billing system — actual invoiced amountsDERIVED from GWh volume assumptions × tariff rate assumptions. User inputs volumes and rates; revenue is calculated.
O&MActual GL amounts from Oracle cost center reportBudget submitted by each department head; roll up to categories. No GL — budget workbook is source.
CapExActual spend from Oracle Projects (capitalised only)Approved project budget from CapEx submission process. Enter by portfolio/category.
Balance SheetOracle Trial Balance at month-endModelled — balance sheet positions derived from operating/capex/financing assumptions. Cash = opening + CF.
FX RateActual billing rate per BOJBudget assumption (fixed for the full year; typically midpoint of BoJ forecast)
DepreciationActual FA module runDerived from FA register + planned CapEx additions
TaxActual provisionDerived from pre-tax income × effective rate

CSV Upload Format for fpa_facts

Use this structure for bulk inserts
ColumnTypeExampleNotes
line_idtextFUEL_REVMust match fpa_dim_line.code — INPUT rows only
version_iduuid(from fpa_versions)Get from SELECT id FROM fpa_versions WHERE code='ACTUALS_2026_03'
period_idinteger(from fpa_dim_period)Get from SELECT id FROM fpa_dim_period WHERE year=2026 AND month=3
valuenumeric12600.00USD 000. Costs entered as POSITIVE numbers.
sourcetextactuals_uploadTag: 'actuals_upload', 'seed', 'manual', 'aop_upload'

Sample SQL — Insert March 2026 Actuals Fuel Revenue:

INSERT INTO fpa_facts (version_id, line_id, period_id, value, source, note)
SELECT
  v.id,
  'FUEL_REV',
  p.id,
  12600.00,
  'actuals_upload',
  'March 2026 Actuals — Billing System export'
FROM fpa_versions v
CROSS JOIN fpa_dim_period p
WHERE v.code = 'ACTUALS_2026_03'
  AND p.year = 2026 AND p.month = 3
ON CONFLICT (version_id, line_id, period_id) DO UPDATE
  SET value = EXCLUDED.value,
      source = EXCLUDED.source,
      note = EXCLUDED.note,
      entered_at = now();

⚠ Note: fpa_facts currently has no PK constraint on (version_id, line_id, period_id). Add this constraint before running upserts to prevent duplicates.
